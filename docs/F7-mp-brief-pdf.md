# F7 — MP Brief export (one-page PDF)

One click → a one-page brief of the top-10 ranked works with evidence
summaries. The artifact an MP's office carries into a district planning
meeting — and the proof for the "non-technical office understands it in
5 minutes" judging criterion.

**Status:** not started. Build after F5 (needs ranks + justifications).

## Approach decision (2026-07-08)

Primary: **print-styled HTML route + browser "Save as PDF"** — zero new
dependencies (matters on the 2GB droplet). The button opens
`/brief` (print CSS, `@page` size A4) and triggers `window.print()`;
the office gets a PDF via the browser's native dialog, and the demo
flourish is identical. Server-side weasyprint (~+150MB image, native pango
libs, more RAM) is the fallback ONLY if judges/organizers require a raw
`.pdf` download link.

## Prerequisites

- [ ] F5 complete (ranked works with justifications)
- [ ] (fallback path only) `weasyprint` in `backend/requirements.txt` +
      `apt-get install -y libpango-1.0-0 libpangocairo-1.0-0
      libgdk-pixbuf-2.0-0 fonts-noto` in `backend/Dockerfile`

## Step-by-step

1. **Template** `backend/app/templates/brief.html` (Jinja2 — fastapi already
   pulls it in): header (constituency, date, submission totals), then top-10
   works — rank, title, ward, one evidence fact, one-line justification —
   then a footer: "Generated by sunwai from N citizen submissions,
   M wards heard, period X–Y." Single page, printable, no color dependence
   (offices print B/W).
2. **Endpoint** `GET /api/brief.pdf` in a new `routers/brief.py`:
   query the same data as `/api/works` (top 10) + totals, render Jinja →
   `weasyprint.HTML(string=html).write_pdf()` → `Response(content=pdf,
   media_type="application/pdf", headers={"Content-Disposition":
   'attachment; filename="sunwai-brief.pdf"'})`.
3. **Optional Gemini executive summary**: one call — "Summarize the top-10
   list in 3 sentences for a district planning meeting" — cache it with the
   same idempotency key trick as F5 justifications.
4. **UI**: "Export MP Brief" button in the header → plain
   `<a href="/api/brief.pdf">`.
5. Redeploy.

## Testing

1. **Golden path**: `curl -o brief.pdf localhost:8082/api/brief.pdf` →
   valid PDF (`file brief.pdf`), opens, exactly 1 page with 10 works.
2. **Content parity**: the 10 works and their ranks match
   `GET /api/works` at the moment of export.
3. **Empty/sparse DB**: with <10 ranked works the brief renders what exists
   without crashing or leaving "None" strings.
4. **Unicode**: ward/demand titles containing Devanagari render correctly in
   the PDF (weasyprint needs a font with Devanagari — add `fonts-noto` to
   the Dockerfile apt line and verify).
5. **Print test**: print it (or B/W preview) — still legible, no dark
   backgrounds.
6. **Timing**: export completes < 5s with warm cache (it's a live demo
   moment — the flourish at the end).
